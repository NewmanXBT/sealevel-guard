# Positioning

## What We Are

An agent-facing Solana security service.

## What We Are Not

- not a generic crypto chatbot,
- not a human-only audit consultancy pitch,
- not a pure per-request x402 demo,
- not a general Rust static analyzer.

## Category

`Security clearing infrastructure for the Solana agent economy`

## Main User

Not the end user first.

The primary user is:

- an integration agent,
- treasury agent,
- deployment agent,
- trading agent,
- or builder agent.

The human operator is the secondary consumer.

## Core Promise

Before an agent acts on Solana, it can ask:

`Is this safe enough to use?`

## Differentiation

### Versus generic audit bots

We are Solana-native, not chain-agnostic.

### Versus static scanners

We emit agent-readable decision artifacts, not only findings.

### Versus x402 pay-per-request APIs

We price by codebase complexity and audit scope, not by raw endpoint calls.

### Versus traditional audits

We are faster, composable, API-native, and designed for machine consumers.

## Positioning Sentence

Sealevel Guard is the security clearing layer that lets Solana agents evaluate
protocol and code risk before they deploy, integrate, trade, or route funds.

## Why The Pricing Story Matters

If we describe the product as:

- `pay per API call`

then we invite the market to compare us with cheap data APIs.

If we describe it as:

- `quoted security work settled through x402`

then we position ourselves as high-value infrastructure with rational margins.
